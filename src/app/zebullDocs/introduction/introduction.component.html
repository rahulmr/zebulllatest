Introduction

Mobull Rest based web services are build on top of Thomson Reuters (Omnesys) REST API which connects to their NEST based OMS and RMS systems. This acts as a wrapper to simplify the complex nature of the original API. Mobull REST provides an easy way to place orders, view order book, watch positions and view funds and balances.

Mobull is built as a complete REST based platform and all input and output parameters are based on JSON unless stated otherwise. The service calls are platform independent and the examples given are Javascript based (AJAX) calls.

Each API call is structured as below


  • -URL to be called
  • -Input Parameters with example JSON set
  • -Response from server with all possible keys
  • -Error messages
  • -Sample JS programs (Optional)
BASE URL

The BASE URL is the common url used as prefix for all API call.

            
Base URL : https://www.zebull.in/rest/MobullService/
GENERATE API KEY

  • User should first login through zebull web application.

  • The API key can be generated through profile settings.

  • The generated API key will be send to the user's registered mail.

  • For authentication and authorization work flow, use that API key.

    • AUTHENTICATION

    The following diagram depicts the usual API authentication flow. After getting the encryption key, user has to generate SHA-256 encrypted key to get User's Session Id. To do this, the user needs to encrypt the combintaion of User id, API key and encryption Key. Sending this to the API will provide the session id. Use the session id in authorization header for all further API calls. If there are any issues found, like unauthorized access or invalid session, user can re-generate session id by using the same set of process described above.


    AUTHORIZATION

    Every request except "authentication" should be appended with Authorization header having Bearer token as below

                    
    Authorization: Bearer userId userSessionId
    Bearer Constant text. Adding this depends on the platform from which the user is going to make the call
    userId User ID used for login
    userSessionId User Session ID which will be received upon successful API authentication
