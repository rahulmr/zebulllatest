Get Session Id

Session Id will be used for rest of the API service calls. To generate the session ID the combination of User Id, API Key (Generated in Zebull web portal) and the encryption Key received in the previous step should be concatenated. The concatenated string should be hashed using SHA-256 algorithm. The sample on the right side provides the implementation example for JavaScript. The same can be done using any other programming language using the respective SHA-256 library.


URL

< BASE URL > + v1/customer/getUserSID

Method

Post

Input Parameters

S.NO KEY VALUE DESCRIPTION
1 userId String User Id User Id
2 userData String User Data Encrypted Data

Note

* Encrypt the data using SHA-256 algorithm.

* The data to be encrypted USER ID, API key and encryption key.

* You can generate API key on zebull web application and you can get encryption key.


Example

       {{'{'}}
       "userId":"abc123",
       "userData":"Encrypted Data" 
       {{'}'}}

Success Response

S.NO PARAMETERS RESPONSE
1 stat String Ok
2 sessionID String Session Id

Error

S.NO PARAMETERS RESPONSE
1 stat String Not_Ok
2 emsg String
  • User does not login. Please login
